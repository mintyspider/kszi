ПРОТОКОЛ ДИФФИ-ХЕЛЛМАНА (основная идея)

Протокол Диффи-Хеллмана позволяет двум сторонам создать общий секретный ключ,
обмениваясь сообщениями по открытому (незащищённому) каналу.

┌─────────────────────────────────────────────────────────────────────────┐
│  Алиса                               Боб                                │
│  ─────                               ───                                │
│  выбирает a (секрет)                 выбирает b (секрет)                │
│  вычисляет A = g^a mod p             вычисляет B = g^b mod p            │
│       │                                     │                           │
│       └─────────── A = g^a mod p ──────────►│                           │
│       ◄─────────── B = g^b mod p ───────────┘                           │
│       │                                     │                           │
│  K = B^a mod p                         K = A^b mod p                    │
│  K = (g^b)^a mod p                     K = (g^a)^b mod p                │
│  K = g^(ab) mod p                      K = g^(ab) mod p                 │
│                                                                         │
│  ОБЩИЙ СЕКРЕТ: K = g^(ab) mod p (совпадает!)                            │
└─────────────────────────────────────────────────────────────────────────┘

ФУНКЦИЯ powerMod() - БЫСТРОЕ ВОЗВЕДЕНИЕ В СТЕПЕНЬ

Обычное возведение в степень: 2^1000 = 2*2*2*... (1000 умножений)
Бинарное возведение в степень: 1000 операций → ~10 операций

Алгоритм (на примере: 2^13 mod 5):

13 в двоичной системе = 1101₂ = 8 + 4 + 0 + 1

Идём по битам справа налево:
─────────────────────────────────────────────────
Бит │  base  │  действие
─────────────────────────────────────────────────
  1 │   2^1  │  result *= 2^1
  0 │   2^2  │  (пропуск)
  1 │   2^4  │  result *= 2^4
  1 │   2^8  │  result *= 2^8
─────────────────────────────────────────────────

result = 2^1 * 2^4 * 2^8 = 2^13 

ТЕСТ МИЛЛЕРА-РАБИНА (проверка простоты)

Вероятностный тест, основанный на свойствах простых чисел:

Для простого числа n можно записать:
n - 1 = d * 2^s, где d - нечётное

Тогда для любого a (1 < a < n) выполняется:
a^d ≡ 1 (mod n) ИЛИ a^(d*2^r) ≡ -1 (mod n) для некоторого r < s

Если для какого-то a это НЕ выполняется → n составное
Если выполняется для многих a → n простое с высокой вероятностью

Вероятность ошибки: 1/4^rounds
При rounds = 40: 1/4^40 ≈ 1/(1.2×10^24) - ничтожно мала

ПЕРВООБРАЗНЫЙ КОРЕНЬ (generator)

Число g называется первообразным корнем по модулю p, если:
{ g^1 mod p, g^2 mod p, ..., g^(p-1) mod p } = {1, 2, ..., p-1}

То есть g "порождает" все числа от 1 до p-1.

Свойство: для любого y (1 < y < p) существует x: g^x ≡ y (mod p)

Проверка: g является первообразным корнем ⇔
для всех простых q | (p-1): g^((p-1)/q) mod p ≠ 1

│ ├── находит все простые множители числа p-1                             │

ПОСЛЕДОВАТЕЛЬНОСТЬ ВЫПОЛНЕНИЯ

main()
  └── simulateExchange()
        │
        ├── ЭТАП 1: Выбор общих параметров
        │     ├── generateRandomPrime(64) → большое простое p
        │     └── findPrimitiveRoot(p) → первообразный корень g
        │
        ├── ЭТАП 2: Генерация ключей
        │     ├── Алиса: generateKeys(p, g)
        │     │     ├── privateKey = случайное (1, p-1)
        │     │     └── publicKey = g^privateKey mod p
        │     └── Боб: аналогично
        │
        ├── ЭТАП 3: Обмен открытыми ключами
        │     ├── Алиса → Боб: publicKey_A
        │     └── Боб → Алиса: publicKey_B
        │
        ├── ЭТАП 4: Вычисление общего секрета
        │     ├── Алиса: secret = publicKey_B^privateKey_A mod p
        │     └── Боб: secret = publicKey_A^privateKey_B mod p
        │
        └── ЭТАП 5: Проверка
              └── if (secret_Alice == secret_Bob) → УСПЕХ

Где используется Диффи-Хеллман:

1. TLS/SSL (HTTPS) - при установке защищённого соединения
2. SSH - для безопасного удалённого доступа
3. IPsec - для VPN-соединений
4. WhatsApp, Telegram - для сквозного шифрования
5. PGP/GPG - для шифрования почты
