1. Класс RSA
-------------------
Класс RSA содержит все алгоритмы для генерации ключей, шифрования и расшифровки.

Переменные:
- random: SecureRandom — используется для генерации случайных чисел.
- smallPrimes: список небольших простых чисел, генерируется функцией generateSmallPrimes для быстрой проверки делимости.

2. Функция generateSmallPrimes(limit)
-------------------
Используется решето Эратосфена для создания списка маленьких простых чисел.
Эти числа применяются для быстрой проверки, делится ли кандидат на маленькие простые, что экономит время перед более сложной проверкой Миллера-Рабина.

3. Функция modPow(base, exponent, mod)
-------------------
Быстрое возведение в степень по модулю (рекурсивно).
- Базовый случай: степень 0 → возвращает 1.
- Базовый случай: степень 1 → возвращает base % mod.
- Рекурсивно возводит в квадрат половину степени и корректирует для нечетных степеней.
Используется для:
- проверки простоты в Миллер-Рабине,
- шифрования и расшифровки в RSA.

4. Функция isPrime(n, iterations)
-------------------
Проверка на простое число с помощью теста Миллера-Рабина.
Алгоритм:
- Проверка чисел меньше 2 и 2,3 напрямую.
- Проверка делимости на маленькие простые числа из smallPrimes.
- Представление n-1 как d * 2^s.
- Повторяем несколько раз: выбирается случайное a, вычисляется x = a^d mod n.
- Если x == 1 или x == n-1 → возможно простое.
- Иначе повторное возведение в квадрат x в цикле.
- Если ни один результат не равен n-1 → число составное.

5. Функция generateRandomPrime(bits)
-------------------
Генерирует случайное простое число:
- Генерируется случайное число candidate заданной битовой длины.
- Если число четное → увеличивается на 1.
- Проверяется isPrime → возвращается, если простое.

6. Функция gcd(a, b)
-------------------
Наибольший общий делитель (НОД) методом Евклида.

7. Функция extendedGCD(a, b)
-------------------
Расширенный алгоритм Евклида.
Возвращает Triple(gcd, x, y), такое что a*x + b*y = gcd.
Используется для нахождения обратного элемента по модулю при генерации закрытой экспоненты d.

8. Функция generateKeys(e)
-------------------
Генерация пары ключей RSA:
- Выбираются два случайных простых числа p и q.
- Вычисляется n = p*q и phi = (p-1)*(q-1).
- Проверяется gcd(e, phi) == 1.
- Вычисляется d как обратное к e по модулю phi с помощью extendedGCD.
- Возвращается объект RSAKeyPair (открытый ключ + закрытый ключ с p,q).

9. Функция encrypt(message, publicKey)
-------------------
Шифрует сообщение с использованием открытой экспоненты:
- ciphertext = message^e mod n.

10. Функция decrypt(ciphertext, privateKey)
-------------------
Расшифровка с использованием Китайской теоремы об остатках (CRT):
- Вычисляются dp = d mod (p-1), dq = d mod (q-1).
- Вычисляется m1 = ciphertext^dp mod p, m2 = ciphertext^dq mod q.
- Используется обратное к p и q для сборки полного результата.
- Итоговое сообщение = (m1*invQ*q + m2*invP*p) mod n.
- CRT ускоряет расшифровку, особенно для больших чисел.

11. Основные структуры данных
-------------------
- PublicKey: хранит e и n.
- PrivateKey: хранит d, n, p, q.
- RSAKeyPair: содержит открытый и закрытый ключ.

12. Функция main()
-------------------
Пошаговое выполнение программы:
1) Ввод открытой экспоненты e или использование значения по умолчанию.
2) Генерация ключей RSA (p, q, n, phi, e, d).
3) Ввод сообщения (целое число).
4) Шифрование сообщения с открытым ключом.
5) Расшифровка сообщения с CRT.

Примечания:
-------------------
- Использование CRT ускоряет расшифровку примерно в 3-4 раза.
- Малые простые числа ускоряют проверку кандидатов при генерации p и q.
- Модульное возведение в степень используется во всех основных математических операциях RSA.
- Все функции реализованы без использования сторонних библиотек для RSA.
